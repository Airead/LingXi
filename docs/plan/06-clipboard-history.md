# F05: 剪贴板历史 - 实现方案

## 详细设计

### 1. 数据库表（复用现有 usage.db）

参考 WenZi `ClipboardMonitor._SCHEMA`：

```sql
CREATE TABLE IF NOT EXISTS clipboard_history (
    id               INTEGER PRIMARY KEY AUTOINCREMENT,
    content_type     TEXT    NOT NULL,          -- 'text', 'image'
    text_content     TEXT    NOT NULL DEFAULT '',-- 文本内容或空串
    timestamp        REAL    NOT NULL,
    source_app       TEXT    NOT NULL DEFAULT '',-- 来源应用显示名
    source_bundle_id TEXT    NOT NULL DEFAULT '',-- 来源应用 bundle id
    image_path       TEXT    NOT NULL DEFAULT '',-- 图片文件名（不含目录）
    image_width      INTEGER NOT NULL DEFAULT 0,
    image_height     INTEGER NOT NULL DEFAULT 0,
    image_size       INTEGER NOT NULL DEFAULT 0, -- PNG 文件字节数
    ocr_text         TEXT    NOT NULL DEFAULT '' -- 图片 OCR 识别文本
);
CREATE INDEX IF NOT EXISTS idx_clipboard_timestamp ON clipboard_history(timestamp DESC);
```

- **去重**：仅与最新条目比较（文本比内容，图片比文件名 hash），不做全局去重（同 WenZi）
- 图片存储到 `~/Library/Application Support/LingXi/clipboard_images/`
- 图片文件名格式：`{timestamp}_{sha256[:12]}.png`（同 WenZi）
- 超大文本（>10KB = 10240 字符）直接跳过不记录（同 WenZi `_MAX_TEXT_LENGTH`）

### 2. ClipboardStore

```
final class ClipboardStore: @unchecked Sendable
```

**常量定义（参考 WenZi）：**
```swift
private let maxTextLength = 10240        // 10 KB，超过则跳过
private let minImageDimension = 4        // 忽略 < 4x4 的图片（追踪像素）
private let pollInterval: TimeInterval = 0.5  // 轮询间隔
```

**密码管理器/隐藏内容过滤（参考 WenZi `_CONCEALED_TYPES`）：**

检测剪贴板是否包含以下任一 PasteboardType，如包含则跳过记录：

```swift
private static let concealedTypes: Set<String> = [
    // nspasteboard.org 标准类型
    "org.nspasteboard.ConcealedType",
    "org.nspasteboard.TransientType",
    "org.nspasteboard.AutoGeneratedType",
    // 密码管理器和文本扩展工具
    "com.agilebits.onepassword",               // 1Password
    "de.petermaurer.TransientPasteboardType",   // TextExpander / Butler (legacy)
    "com.typeit4me.clipping",                   // TypeIt4Me
    "Pasteboard generator type",                // Typinator
    "net.antelle.keeweb",                       // KeeWeb
]

private static func isConcealed(_ pasteboard: NSPasteboard) -> Bool {
    guard let types = pasteboard.types else { return false }
    let typeStrings = Set(types.map(\.rawValue))
    return !concealedTypes.isDisjoint(with: typeStrings)
}
```

**监听机制：**
- `DispatchSourceTimer` 每 0.5s 在 `monitorQueue` 上轮询 `NSPasteboard.general.changeCount`
- 检测到变化时：
  1. 检查 concealed types → 跳过
  2. 检查 `skipNextChange` 标记（LingXi 自身写入时设置）→ 跳过
  3. 获取来源应用：`DispatchQueue.main.sync { NSWorkspace.shared.frontmostApplication }`
  4. 按优先级读取内容

**内容类型优先级（参考 WenZi，PNG → Text → TIFF）：**

```
PNG 优先：用户复制真实图片时（浏览器、图片编辑器）设置 PNG
  → 检查 PNG 数据，若保存成功则结束
  → PNG 失败（如图片太小 < 4x4）则 fallthrough

Text 其次：最常见的复制类型
  → 检查纯文本，trim 后非空 && <= 10KB 则记录
  → > 10KB 则跳过不记录

TIFF 最后：macOS 常将富文本的 TIFF 渲染放入剪贴板
  → 仅在无 PNG 且无 Text 时才作为图片记录
  → 避免每次文本复制都产生图片条目
```

**图片存储（参考 WenZi `_save_image`）：**
- 统一转为 PNG 存储
- 文件名：`{unixTimestamp}_{sha256(pngData).prefix(12)}.png`
- 存储前检查文件是否已存在（hash 碰撞时加后缀 `_1`, `_2`...）
- 忽略宽高均 < 4px 的图片（追踪像素等伪影）

**去重逻辑（参考 WenZi `_add_entry`）：**
- 仅与内存中最新条目比较：
  - 文本：`cachedItems.first?.textContent == newText` → 跳过
  - 图片：通过 hash 产生相同文件名 → 跳过
- 不做全局去重，这样旧内容再次复制会产生新记录（符合用户预期）

**图片保存后触发 OCR（参考 WenZi `_add_image_entry` → `_run_ocr`）：**
- 图片宽高均 >= 100px 时，异步提交到 OCR 队列
- OCR 完成后：`UPDATE clipboard_history SET ocr_text = ? WHERE image_path = ?`
- 同步更新内存缓存中对应条目的 `ocrText` 字段，`version += 1`

**容量清理：**
- 每次新增后检查：`DELETE FROM clipboard_history WHERE id NOT IN (SELECT id FROM clipboard_history ORDER BY timestamp DESC LIMIT ?)`
- 同时清理被删除条目的图片文件

**版本计数（参考 WenZi `_version`）：**
- 每次内存缓存变更时 `version += 1`
- 用于 Provider 的空查询缓存失效检测

**线程安全：** NSLock 保护 `cachedItems` 数组和 `version`

**关键方法：**
- `startMonitoring()` / `stopMonitoring()`
- `var items: [ClipboardItem]` — 线程安全的缓存访问
- `var version: Int` — 单调递增版本号
- `delete(itemId:)` — 删除记录及关联图片
- `writeToClipboard(itemId:) -> Bool` — 将历史条目写回系统剪贴板（带 concealed 标记防自记录）
- `setCapacity(_:)` — 更新容量上限

**ClipboardItem 结构：**
```swift
struct ClipboardItem: Sendable {
    let id: Int
    let contentType: ContentType  // .text, .image
    let textContent: String       // 文本内容，图片时为空串
    let sourceApp: String
    let sourceBundleId: String
    let imagePath: String         // 文件名，文本时为空串
    let imageWidth: Int
    let imageHeight: Int
    let imageSize: Int            // PNG 字节数
    let ocrText: String           // OCR 识别文本，无则空串
    let timestamp: TimeInterval
}
```

### 3. ClipboardHistoryProvider

```
final class ClipboardHistoryProvider: SearchProvider, @unchecked Sendable
```

- **空查询缓存（参考 WenZi `_CACHE_TTL = 10.0`）：**
  - 缓存上次空查询结果，TTL 10 秒
  - 失效条件：`store.version` 变化 或 超过 10 秒（相对时间字符串会过时）
- **空查询**：返回按时间倒序的所有条目（最多 maxResults 条）
- **非空查询**：使用 `scoredResults()` 对 textContent 做模糊匹配；图片条目搜索 OCR 文本 + 来源应用名（参考 WenZi `fuzzy_match_fields` 同时匹配多个字段）
- 每个 SearchResult：
  - `itemId`: `"clipboard:\(item.id)"`
  - `name`: 文本预览（截断 80 字符）/ "Image: {width}×{height} ({fileSize})"
  - `subtitle`: "{来源应用} {相对时间}"
  - `resultType`: `.clipboard`
  - `icon`: 文本用来源应用图标 / 图片用缩略图（从磁盘加载）
  - `modifierActions[.command]`: "Copy to clipboard"（仅复制，不粘贴）

### 5. 粘贴模拟

放在 `PanelManager.swift` 中作为私有辅助。WenZi 使用 `osascript` + AppleScript，但 Swift 原生项目推荐用 CGEvent：

```swift
private func simulatePaste() {
    // CGEvent 方式（需要辅助功能权限）
    guard AXIsProcessTrusted() else { return }
    let source = CGEventSource(stateID: .hidEventState)
    let keyDown = CGEvent(keyboardEventSource: source, virtualKey: 0x09, keyDown: true)  // V key
    let keyUp = CGEvent(keyboardEventSource: source, virtualKey: 0x09, keyDown: false)
    keyDown?.flags = .maskCommand
    keyUp?.flags = .maskCommand
    keyDown?.post(tap: .cghidEventTap)
    keyUp?.post(tap: .cghidEventTap)
}
```

延迟 50ms 后调用（参考 WenZi `time.sleep(0.05)`），让目标窗口获得焦点后再粘贴。

### 6. 相对时间格式化（参考 WenZi `_format_time_ago`）

```swift
static func relativeTime(from timestamp: TimeInterval) -> String {
    let delta = Date().timeIntervalSince1970 - timestamp
    switch delta {
    case ..<60:    return "just now"
    case ..<3600:  return "\(Int(delta / 60))m ago"
    case ..<86400: return "\(Int(delta / 3600))h ago"
    default:       return "\(Int(delta / 86400))d ago"
    }
}
```

### 7. OCR 图片文字识别

**触发条件（参考 WenZi `_MIN_OCR_DIM = 100`）：**
- 图片宽高均 >= 100px 时自动触发 OCR
- 在后台队列异步执行，不阻塞监听轮询

**实现方式 — 使用 macOS Vision 框架（参考 WenZi `ocr.py`）：**

```swift
import Vision

func recognizeText(imagePath: String) -> String {
    let url = URL(fileURLWithPath: imagePath)
    guard let handler = try? VNImageRequestHandler(url: url) else { return "" }

    let request = VNRecognizeTextRequest()
    request.recognitionLevel = .fast  // Fast 级别支持 30+ 语言含中文
    request.recognitionLanguages = ["zh-Hans", "zh-Hant", "en-US"]

    try? handler.perform([request])

    guard let results = request.results else { return "" }
    return results.compactMap { observation in
        observation.topCandidates(1).first?.string
    }.joined(separator: "\n")
}
```

**关键参数：**
- `recognitionLevel = .fast`：覆盖 30+ 语言（含简繁中文），`.accurate` 仅支持 6 种拉丁语言
- `recognitionLanguages = ["zh-Hans", "zh-Hant", "en-US"]`：简中、繁中、英文

**存储与搜索：**
- OCR 结果存入 `clipboard_history` 表的 `ocr_text` 字段
- OCR 完成后更新数据库和内存缓存
- 搜索时，图片条目同时匹配 OCR 文本（参考 WenZi 在 `fuzzy_match_fields` 中包含 `entry.ocr_text`）

**异步执行（参考 WenZi 的 `ThreadPoolExecutor`）：**
- 使用 `DispatchQueue(label: "io.github.airead.lingxi.ocr", qos: .utility)` 后台执行
- OCR 完成后更新数据库和刷新缓存版本号
