# LingXi 总体开发计划

## 项目定位

LingXi 是一个 macOS 启动器应用，类似 Alfred / Raycast，通过全局快捷键快速呼出搜索框，实现应用启动、文件搜索等功能。

## 技术选型

| 层面 | 技术 | 说明 |
|------|------|------|
| UI 框架 | SwiftUI | 主要 UI 开发 |
| 窗口管理 | AppKit (NSPanel) | 实现无标题栏浮动面板 |
| 全局快捷键 | CGEvent / NSEvent | 监听全局热键 |
| 搜索引擎 | NSMetadataQuery | 复用 Spotlight 索引搜索应用和文件 |
| 最低支持版本 | macOS 14+ | 使用较新的 SwiftUI API |

## 功能模块

### 阶段一：基础架构

1. **菜单栏应用** — 常驻菜单栏图标，隐藏 Dock 图标，不显示默认窗口
2. **全局快捷键** — 注册全局热键（如 `⌥ Space`），用于呼出/隐藏搜索面板
3. **浮动搜索面板** — 无标题栏、居中浮动的 NSPanel 窗口，点击外部自动隐藏

### 阶段二：搜索 UI

4. **搜索框** — 输入框 + 实时搜索，支持防抖
5. **结果列表** — 展示搜索结果，显示图标和名称
6. **键盘导航** — 上下方向键选择结果，回车执行，Esc 关闭

### 阶段三：搜索功能

7. **应用搜索** — 搜索已安装应用程序并启动
8. **文件搜索** — 基于 Spotlight 索引搜索文件并打开
9. **搜索结果排序** — 按匹配度和使用频率排序

### 阶段四：核心体验提升

10. **模糊匹配增强（F01）** — 首字母缩写、散序字符匹配，四级评分体系
11. **使用频率学习（F02）** — 记录选择行为，按查询词关联频率加权排序
12. **异步增量搜索（F03）** — 多源并行、增量展示、generation 追踪防旧覆新

### 阶段五：数据源扩展

13. **计算器源（F04）** — 自动识别数学表达式，结果复制到剪贴板
14. **剪贴板历史（F05）** — 监听剪贴板变化，支持文本/图片/文件，快速粘贴
15. **文件搜索（F06）** — 基于 Spotlight 索引，支持 Finder 中显示
16. **系统命令源（F07）** — 锁屏、休眠、切换深色模式等内置系统操作
17. **书签搜索（F08）** — 读取 Safari/Chrome 书签，浏览器中打开

### 阶段六：交互增强

18. **修饰键动作（F09）** — Cmd+Enter/Alt+Enter 执行不同动作，按住修饰键动态提示
19. **Tab 补全（F10）** — 路径补全、前缀激活等源级补全逻辑
20. **查询历史（F11）** — 空输入时 ↑↓ 浏览历史查询
21. **Quick Look 预览（F12）** — 空格键预览文件，与 Finder 体验一致

### 阶段七：可扩展性

22. **插件系统（F13）** — Bundle 形式插件，标准协议，进程隔离
23. **Universal Action（F14）** — 选中文本触发上下文动作（搜索、翻译、编解码等）

### 阶段八：体验打磨

24. **UI 美化（F15）** — 毛玻璃效果、动画过渡、匹配字符高亮、深色/浅色适配
25. **设置界面（F16）** — 快捷键自定义、源管理、开机自启等偏好设置
26. **快捷键可配置（F17）** — 录制组件、冲突检测、多功能独立快捷键

> 详细需求见 [04-feature-requirements.md](04-feature-requirements.md)

## 架构设计

```
LingXi/
├── App/                    # 应用入口、菜单栏、生命周期管理
├── UI/                     # SwiftUI 视图（搜索框、结果列表等）
├── Panel/                  # NSPanel 浮动窗口管理
├── HotKey/                 # 全局快捷键注册与监听
├── Search/                 # 搜索引擎（应用搜索、文件搜索）
├── Models/                 # 数据模型（搜索结果、设置等）
└── Utils/                  # 工具类
```

## 开发原则

- **小步迭代**：每一步都可独立运行和验证
- **测试先行**：每个功能模块配套测试用例
- **最小可用**：先实现核心功能，再逐步增强
