# 阶段二：搜索 UI — 分步计划

## 目标

将浮动面板从占位文本升级为完整的搜索 UI：输入框、结果列表、键盘导航。

---

## Step 1：搜索输入框

**改动：**
- 新建 `UI/SearchViewModel.swift`，包含 `@Published var query: String`
- 将 `PanelContentView` 中的占位文本替换为 `TextField`，绑定 `query`
- 面板呼出时自动聚焦输入框（通过 `@FocusState`）
- 面板隐藏时清空 `query`（在 `PanelManager.hide()` 中触发）

**验证：**
- [ ] 按 `⌥ Space` 呼出面板，光标自动在输入框中
- [ ] 可以正常输入文字
- [ ] 隐藏面板再重新呼出，输入框内容已清空

---

## Step 2：静态结果列表 + 模拟数据

**改动：**
- 新建 `Models/SearchResult.swift`，定义 `SearchResult` 模型（id、icon、name、subtitle）
- `SearchViewModel` 中添加模拟数据集和 `@Published var results: [SearchResult]`
- 输入文本变化时，对模拟数据按名称进行简单过滤
- 在输入框下方添加结果列表视图，每行显示图标 + 名称 + 副标题
- 面板高度根据结果数量动态调整（无结果时只显示输入框，有结果时展开）

**验证：**
- [ ] 输入文字后，下方出现匹配的模拟结果
- [ ] 清空输入框，结果列表消失，面板回到只有输入框的高度
- [ ] 结果行显示图标、名称、副标题

---

## Step 3：键盘导航

**改动：**
- `SearchViewModel` 中添加 `@Published var selectedIndex: Int`
- 拦截键盘事件：↑ / ↓ 方向键移动选中项，回车执行选中项
- 选中行高亮显示，列表自动滚动到可见区域
- 回车键执行选中项（此阶段仅打印日志）
- Esc 关闭面板（已有）

**验证：**
- [ ] 输入文字出现结果后，按 ↓ 键高亮向下移动
- [ ] 按 ↑ 键高亮向上移动，到顶部不越界
- [ ] 按 ↓ 到底部不越界
- [ ] 按回车键，Xcode 控制台打印选中项信息
- [ ] Esc 仍可关闭面板

---

## 完成标准

阶段二完成后，应用表现为：
1. 按 `⌥ Space` 弹出面板，光标自动在搜索框中
2. 输入文字后实时显示匹配的模拟结果
3. 方向键上下选择结果，回车打印选中项
4. Esc 或点击外部关闭面板，再次呼出时输入框已清空
